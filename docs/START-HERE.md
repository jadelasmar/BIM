Rules:
- Keep replies short.
- Focus on implementation.
- BIM Nexus is an internal IT operations platform.
- BIM Stock is the active implemented module.
- Do not add accounting, invoicing, payment, or ticketing/Tasklogger replacement features.
- Do not explain every change.
- Summarize only files changed, important decisions, commands run, and blockers.
- Read relevant code before editing.
- Keep BIM modular.
- Work one session/task at a time.
- Use available plugins/tools when relevant.
- For UI/design tasks, use Figma when a usable Figma URL, screenshot, or generated code is provided.
- For security tasks, use Codex Security if available.
- Use GitHub if relevant for repo, issue, PR, or version-control tasks.
- Update docs when behavior, workflow, roadmap, or structure changes.
