## Operational Workflows

Receiving:
- internal stock-entry records
- not purchasing or accounting
- can store supplier, date, optional reference text, notes, and received items
- completing receiving should create/update ProductUnit records
- should generate stock movement history later

Delivery:
- internal stock-exit/dispatch records
- not invoices
- can store client/customer, receiver, deliverer, signature details, notes, and delivered items
- completing delivery should update ProductUnit status
- should generate stock movement history later

Documents:
- non-stock paperwork module
- examples: installation reports, maintenance forms, signed scans, photos, PDFs
- should not affect stock unless explicitly tied to an inventory workflow

## Later Enhancements

- printable receiving/delivery documents

## Out Of Scope
